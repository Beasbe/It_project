# Документация по развертыванию

## Обзор

Этот проект на Next.js настроен для развертывания на GitHub Pages, как указано в скриптах `package.json` и поле `homepage`. Его также можно развернуть на других платформах, таких как Vercel, Netlify или собственный сервер. Этот документ охватывает процесс развертывания, требования и лучшие практики.

## Предварительные требования

- **Node.js и Yarn**: Убедитесь, что установлены Node.js (рекомендуется v18+) и Yarn. Проект использует Yarn (видно по `yarn.lock`).
- **Репозиторий GitHub**: Для развертывания на GitHub Pages проект должен быть в репозитории GitHub.
- **Зависимости**: Выполните `yarn install` для установки всех пакетов.
- **Артефакты сборки**: Проект собирается в директорию `out/` для статического экспорта.

## Методы развертывания

### 1. GitHub Pages (Основной метод)
Проект настроен для статического экспорта и развертывания на GitHub Pages.

#### Шаги:
1. **Сборка проекта**:
   - Выполните `yarn build`. Это создает production-сборку.
   - Для статического экспорта (требуется для GitHub Pages) убедитесь, что в `next.config.ts` есть `output: 'export'`, если не установлено. (Примечание: Проверьте `next.config.ts` – если отсутствует, добавьте.)

2. **Скрипт развертывания**:
   - В `package.json` включены:
     - `"predeploy": "next build"`,
     - `"deploy": "touch out/.nojekyll && gh-pages -d out"`,
     - `"deploy:gh": "yarn build && yarn deploy"`.
   - Выполните `yarn deploy:gh`.
   - Это использует пакет `gh-pages` для推送 директории `out/` в ветку `gh-pages` вашего репозитория GitHub.

3. **Конфигурация**:
   - Установите `homepage` в `package.json` на URL GitHub Pages (например, "https://username.github.io/repo/").
   - В настройках репозитория GitHub включите GitHub Pages из ветки `gh-pages`.
   - Файл `.nojekyll` предотвращает обработку сайта Jekyll.

4. **Доступ к сайту**:
   - Посетите `https://username.github.io/repo/` (замените на ваши данные).

#### Примечания:
- Это развертывает статическую версию сайта. Серверные функции (например, маршруты API) не будут работать, если не использовать другой метод хостинга.
- Если используются динамические функции Next.js, рассмотрите другие платформы.

### 2. Vercel (Рекомендуется для Next.js)
Vercel оптимизирован для Next.js и обрабатывает статические, серверless и edge-функции.

#### Шаги:
1. **Установка Vercel CLI**: `npm i -g vercel` (или используйте Yarn).
2. **Вход**: Выполните `vercel login`.
3. **Развертывание**:
   - Из корня проекта: `vercel`.
   - Следуйте подсказкам для связи с вашим аккаунтом Vercel и репозиторием.
   - Для production: `vercel --prod`.
4. **Пользовательский домен**: Настройте в панели Vercel.

#### Преимущества:
- Автоматические сборки, предпросмотры для PR.
- Поддержка маршрутов API и рендеринга на стороне сервера.
- Доступен бесплатный тариф.

### 3. Netlify
Похож на Vercel, отличен для статических сайтов.

#### Шаги:
1. **Подключение репозитория**: Свяжите ваш репозиторий GitHub в панели Netlify.
2. **Команда сборки**: Установите `next build && next export` (для статического) или просто `next build` (для серверless).
3. **Директория публикации**: `out/` для статического экспорта.
4. **Развертывание**: Netlify обрабатывает непрерывное развертывание при пушах.

### 4. Собственный сервер или другие платформы
- **Статический экспорт**: Выполните `next export` после сборки, затем обслуживайте директорию `out/` с любого статического хоста (например, AWS S3, Firebase Hosting).
- **Развертывание на сервере**: Для полных функций Next.js разверните на сервере Node.js (например, Heroku, DigitalOcean):
  - Сборка с `next build`.
  - Запуск с `next start`.
  - Используйте PM2 или подобное для production.

## Переменные окружения
- Если используются файлы `.env`, убедитесь, что они установлены на платформе хостинга (например, в панелях Vercel/Netlify).
- Для чувствительных данных, таких как API-ключи, никогда не коммитьте их в Git.

## Интеграция CI/CD
- **GitHub Actions**: Директория `.github/` может содержать workflow. Добавьте один для автоматических сборок/развертываний при пушах.
- Пример: Используйте actions/setup-node и запускайте скрипты развертывания.

## Устранение неисправностей
- **Ошибки 404 при обновлении**: Для SPA на GitHub Pages убедитесь, что маршрутизация обработана (экспорт Next.js управляет этим).
- **Сбои сборки**: Проверьте совместимость версии Node, пределы памяти.
- **HTTPS**: Большинство платформ принуждают HTTPS; GitHub Pages тоже.
- **Большие сборки**: Оптимизируйте изображения и разделяйте код.

## Лучшие практики
- **Контроль версий**: Развертывайте из стабильной ветки (например, main).
- **Тестирование**: Запускайте `yarn lint` и тесты перед развертыванием.
- **Мониторинг**: Используйте аналитику платформы или интегрируйте инструменты вроде Google Analytics.
- **Откат**: Храните предыдущие сборки или используйте версионированные развертывания.

Для дополнительных деталей обратитесь к документации по развертыванию Next.js: [https://nextjs.org/docs/deployment](https://nextjs.org/docs/deployment).
