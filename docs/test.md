# Документация по тестированию

## Обзор

Тестирование критически важно для обеспечения надежности, функциональности и производительности приложения Next.js "Пятилетка Tech Solutions". Оно помогает выявлять ошибки на ранних этапах, проверять поведение компонентов и поддерживать качество кода во время разработки и рефакторинга.

В настоящее время в проекте отсутствует выделенная настройка тестирования, поскольку нет файлов тестов или скриптов в `package.json` помимо линтинга. Этот документ описывает, как реализовать тестирование, рекомендуемые инструменты и лучшие практики. Мы рекомендуем начинать с unit- и integration-тестов для компонентов и страниц.

## Текущее состояние тестирования

- **Тесты не реализованы**: В проекте нет файлов тестов (например, в директории `__tests__`) или настроенных тест-раннеров.
- **Линтинг**: ESLint настроен через `"lint": "eslint"` в `package.json`, который проверяет стиль кода и потенциальные ошибки, но не заменяет функциональные тесты.
- **Возможности**: Компоненты вроде `Hero.tsx` (с хуками и анимациями) идеальны для тестирования взаимодействий, изменений состояния и рендеринга.

Если тесты будут добавлены, обновите этот документ отчетами о покрытии и конкретными тест-кейсами.

## Рекомендуемые инструменты тестирования

- **Jest**: Для запуска тестов, утверждений и мокинга. Это дефолт для Next.js.
- **React Testing Library**: Для тестирования React-компонентов способом, имитирующим взаимодействия пользователя.
- **Cypress или Playwright**: Для end-to-end (E2E) тестирования полных пользовательских потоков.
- **Поддержка TypeScript**: Используйте `@types/jest` и `ts-jest` для тестов на TypeScript.

### Установка

Добавьте зависимости, выполнив:

```bash
yarn add --dev jest @testing-library/react @testing-library/jest-dom ts-jest @types/jest
```

Обновите `package.json` скриптом для тестов:

```json
"scripts": {
  "test": "jest",
  "test:watch": "jest --watch"
}
```

Создайте файл `jest.config.ts` в корне:

```typescript
import type { Config } from 'jest';

const config: Config = {
  preset: 'ts-jest',
  testEnvironment: 'jsdom',
  setupFilesAfterEnv: ['<rootDir>/jest.setup.ts'],
  moduleNameMapper: {
    '\\.(css|less|scss|sass)$': 'identity-obj-proxy',
  },
};

export default config;
```

Создайте `jest.setup.ts`:

```typescript
import '@testing-library/jest-dom';
```

## Типы тестов

### 1. Unit-тесты
- **Назначение**: Тестирование отдельных функций, хуков или утилит в изоляции.
- **Пример**: Тестирование пользовательского хука в `src/hooks/`.

Пример теста для простой функции (гипотетический `src/lib/utils.ts`):

```typescript
// src/lib/utils.test.ts
import { someFunction } from './utils';

test('someFunction adds numbers', () => {
  expect(someFunction(2, 3)).toBe(5);
});
```

### 2. Integration-тесты
- **Назначение**: Тестирование взаимодействия компонентов, включая рендеринг и события пользователя.
- **Пример**: Тестирование компонента `Hero`.

Создайте `src/components/Hero.test.tsx`:

```typescript
import { render, screen, fireEvent } from '@testing-library/react';
import Hero from './Hero';

test('renders Hero component with title', () => {
  render(<Hero />);
  expect(screen.getByText('ПЯТИЛЕТКА')).toBeInTheDocument();
});

test('handles hover state', () => {
  render(<Hero />);
  const container = screen.getByTestId('hero-container'); // Add data-testid in component
  fireEvent.mouseEnter(container);
  // Assert changes, e.g., class additions
});
```

### 3. End-to-End (E2E) тесты
- **Назначение**: Симуляция полных пользовательских путей по страницам.
- **Инструмент**: Установите Cypress: `yarn add --dev cypress`.
- **Настройка**: Добавьте `"cypress": "cypress open"` в скрипты.
- **Пример**: Тестирование навигации с главной страницы на about.

## Запуск тестов

- **Запуск всех тестов**: `yarn test`.
- **Режим наблюдения**: `yarn test:watch` для разработки.
- **Покрытие**: Добавьте `--coverage` в скрипт тестов и просмотрите отчеты в `coverage/`.

## Лучшие практики

- **Покрытие тестов**: Стремитесь к 80%+ покрытию на критических путях (компоненты, страницы, API).
- **Мокинг**: Мокайте внешние зависимости (например, API) для изоляции тестов.
- **Интеграция CI**: Добавьте workflow GitHub Actions для запуска тестов на pull requests.
  Пример workflow в `.github/workflows/test.yml`:

```yaml
name: Run Tests
on: [pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with: { node-version: 18 }
      - run: yarn install
      - run: yarn test
```

- **Тестирование доступности**: Используйте `jest-axe` для проверки проблем a11y.
- **Производительность**: Избегайте перетестирования; фокусируйтесь на областях с высоким воздействием.
- **Поддержка**: Поддерживайте тесты в актуальном состоянии с изменениями кода. Рефакторьте нестабильные тесты.

## Потенциальные тест-кейсы для этого проекта

- **Компонент Hero**: Проверьте, что анимации срабатывают при наведении, кнопка прокручивает правильно (мок window.scrollBy).
- **ContactForm**: Тестируйте валидацию формы, отправку (мок fetch).
- **Страницы**: Убедитесь в правильной композиции компонентов (например, главная страница рендерит все разделы).
- **Маршруты API** (если добавлены): Тестируйте обработчики с supertest или Jest.
- **Темы**: Тестируйте переключения светлого/темного режимов.

Регулярно запускайте и расширяйте тесты для повышения надежности проекта. Для дополнительной информации о тестировании в Next.js см.: [https://nextjs.org/docs/testing](https://nextjs.org/docs/testing).
